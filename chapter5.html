Chapter 5: Programming Concepts and Logic

Introduction

Every computerized device needs instructions to perform tasks. These instructions are written in a programming language. Programs are sets of instructions that tell a computer what actions to perform, and when combined, they make up software. The computer doesn't understand programming languages directly; instead, it requires a language translator such as an assembler, compiler, or interpreter to convert high-level languages into machine language.

The program written by the programmer is called the source program. After being translated, it becomes a machine-understandable object program.

Qualities of Good Programs

  • Understandability: The program should be easy to understand.
  • Correctness: The program must be error-free.
  • Reliability: The program should perform its function consistently and correctly.
  • User Interface: It should be easy to use and navigate.
  • Portability: The program should be flexible enough to run on different systems.

Programming Languages

Programming languages are used to communicate instructions to a computer. These languages consist of a set of keywords, variables, operators, and other elements such as loops and conditions that help in forming instructions.

The process of writing programs is called programming, and the person who writes programs is called a programmer.

Types of Programming Languages

Programming languages can be categorized into low-level and high-level languages.

Low-level Language

Low-level languages are machine-dependent, meaning programs written for one type of system cannot be run on another system. Programming in low-level languages is more difficult and time-consuming.

Types of Low-level Languages:

  • Machine-level Language (1GL): Uses binary code (0’s and 1’s), understood directly by the processor, but difficult to write and debug.
  • Assembly Language (2GL): Uses mnemonics like ADD, SUB, etc., which are easier to understand than binary code but still require knowledge of the system's architecture.

High-level Language

High-level languages are closer to human languages (like English), which makes them easier to use for writing and debugging programs. They are machine-independent, meaning programs can run on different processors without modification.

Types of High-level Languages:

  • Procedural-oriented Languages (3GL): Focuses on the procedure (steps) to be followed, e.g., C, FORTRAN.
  • Problem/ Object-oriented Languages (4GL): Focuses on data rather than procedures and supports data encapsulation, inheritance, etc., e.g., Java, C++.
  • Natural Languages (5GL): Uses simple, natural statements for communication, still in development stage, e.g., PROLOG.

Language Translators/Processors

To convert programs written in assembly or high-level languages into machine language, we use language translators:

  • Assembler: Converts assembly language into machine language.
  • Compiler: Converts the entire program from high-level language to machine language at once (e.g., C, C++).
  • Interpreter: Converts one line or statement at a time and executes it immediately (e.g., BASIC, PHP).

Syntax and Semantics

In programming, syntax refers to the rules that govern the structure of statements in a program. Semantics refers to the meaning of individual statements or symbols used in the program.

Example in QBasic:

Print "This is programming";

In this example, Print is the semantics, and the entire line is the syntax.

Errors (BUGS)

Errors in a program are called bugs, and the process of removing them is called debugging. There are three types of errors:

  • Syntax Error: Errors due to incorrect syntax (e.g., missing semicolons).
  • Logical Error: Errors due to incorrect logic (e.g., incorrect formula for calculation).
  • Runtime Error: Errors that occur during program execution (e.g., insufficient memory).

Common Terminology in Programming

Data Types: Specifies the type of data used in a program, e.g., Numeric, Character.

Operators: Special symbols that represent actions (e.g., +, -, *, /).

Data Types

Data types specify the type of data that can be used in a program. Some common types are:

  • Numeric: Represents positive and negative whole numbers, including zero.
  • Character: Includes alphanumeric values and special symbols.

Example of Numeric:

int age = 25; (where int is a numeric data type).

Example of Character:

char letter = 'A'; (where char is a character data type).

Variable and Constant

Variables and constants are used to store values during program execution.

  • Variable: An entity that holds values which can change during execution.
  • Constant: An entity that holds a value that does not change throughout program execution.

Example of Variable:

int count = 10; (where count is a variable).

Example of Constant:

const int MAX_VALUE = 100; (where MAX_VALUE is a constant).

Operands, Operators, and Operations

Operands are the values that operators act on to perform operations.

Example operation: S = A + B; where A and B are operands, and + is an operator.

The operation itself is called Addition.

Types of Operators:

  • Arithmetic operators: +, -, *, /
  • Relational operators: <, >, <=, >=, ==
  • Logical operators: AND, OR, NOT

Example of Arithmetic Operator:

int result = 10 + 5; (where + is the arithmetic operator).

Library Functions and User-defined Functions

Functions are blocks of code designed to perform specific tasks. There are two types:

  • Library Functions: Predefined functions available in a programming language's library.
  • User-defined Functions: Functions that are created by the user to perform specific tasks.

Examples of Library Functions:

printf("Hello World!"); (a common library function for printing output).
scanf("%d", &x); (a library function for taking input).

Example of a User-defined Function:

int add(int a, int b) { return a + b; } (a simple user-defined function to add two numbers).

Input/Output Functions

Input/Output (I/O) functions are used to read data from the user and display output. Some common I/O functions include:

  • printf(): Used to display output in the console. Example: printf("My School");
  • scanf(): Used to take input from the user. Example: scanf("%d", &x);

Program Design Tools

There are several tools used for designing programs, such as:

  • Algorithm: A step-by-step procedure to solve a specific problem.
  • Flowchart: A graphical representation of the steps in a program.
  • Pseudocode: A simplified version of the program written in plain language.

Algorithm Rules

  • It should execute output after a finite number of steps.
  • It should not depend on any specific programming language.
  • It should terminate after a finite number of steps.

Flowchart

It is a diagrammatic or pictorial representation of the steps involved in an algorithm for solving a particular problem. There are two types of flowcharts:

a. Program Flowchart

It is a type of flowchart which focuses on a specific problem. This type of flowchart portrays only a single task (logic) at a time.

Symbols used in Flowchart:

  • Start/End: Represented by an oval symbol
  • Process: Represented by a rectangle symbol
  • Decision: Represented by a diamond symbol
  • Input/Output: Represented by a parallelogram symbol
  • Arrow: Represented by lines indicating the flow of control

Examples of Flowchart:

  • Flowchart to calculate simple interest
  • Flowchart to find the greatest number among two numbers

b. System Flowchart

It is a type of flowchart that shows the logical flow of a program and its collection. It shows the overview of the data flow and sequence of operations in a system.

Symbols used in System Flowchart:

  • System processes: Represented by rectangular boxes
  • Inputs and outputs: Represented by parallelograms
  • Data stores: Represented by two horizontal lines
  • Arrows: Indicate the flow of data

C Programming Language

The C programming language is a high-level programming language developed by Dennis Ritchie at AT&T's Bell Laboratories in the 1970s. It is a very powerful language, used for both system programming and application programming, and often known as a "middle-level" language.

Features of C Language

  • Supports both high-level and low-level programming
  • Commonly used for system programming and application programming
  • Offers fast and efficient program development
  • Has powerful tools and keywords
  • Uses its own text editor and compiler
  • Supports structured programming
  • Portable across different systems
  • Has a powerful set of library functions
  • Flexible in programming

Advantages and Disadvantages of C Language

Advantages:

  • Suitable for system programming
  • Easy to interact with hardware
  • Easy to learn
  • Compact and efficient code
  • Secure program code

Disadvantages:

  • No runtime checking
  • No strict data type checking
  • Debugging becomes complex as the program grows
  • Requires semicolons for each statement

C Language Terminology

Define the Following Terms

a) Editor

The editor is the area where you type the program. While C language has its own text editor, any text editor can also be used.

b) Pre-processor

A pre-processor is a set of complex codes that instruct the machine to understand specific keywords. For example, #include tells the compiler to use header files like "stdio.h".

c) Compiler

A compiler converts source code written in a high-level programming language into machine-readable code (object code).

d) Linker

A linker is software that links library files with object code to create an executable file (with a .exe extension).

Fundamentals of C

Character Set

The character set is the collection of alphabets, digits, whitespace characters, and special symbols used in programming. C uses the following character set:

  • Alphabets: A-Z, a-z
  • Digits: 0-9
  • Special Symbols: ~ ! @ # $ % ^ & * ( ) - _ + = | \ { } [] : ; " ' <> , . ? /

Comment

Comments are non-executable text in a program used to provide explanations for other developers or programmers. There are two types of comments:

  • Single-line comment: // Comment text here
  • Multi-line comment: /* Comment text here */

C Tokens

The set of identifiers, keywords, constants, variables, and operators used in C programming are called C Tokens.

Identifiers

Identifiers are names given to various program elements such as variables, functions, and arrays.

Keywords

Keywords are reserved words in C that cannot be used as identifiers. Examples: int, void, if, else, while, for.

Data Types in C

  • int: Represents integer values (whole numbers), uses 2 bytes of memory.
  • float: Represents floating-point values (decimal numbers), uses 4 bytes of memory.
  • char: Represents a single character, uses 1 byte of memory.
  • void: Represents an absence of value.

Constants and Variables

Constants are fixed values, while variables are used to store data that can change during program execution.

Operators

Operators are symbols used to perform operations on variables and constants. Examples include arithmetic, relational, and logical operators.

Escape Sequences

  • \n: New line
  • \t: Horizontal tab
  • \0: Null character
  • \\: Backslash

Library Functions

Functions are blocks of code that can be used to perform a task. Examples: printf(), scanf(), strlen().

Understanding Arrays in C Programming

In C programming, an array is a collection of variables of the same type, stored in contiguous memory locations. Arrays allow you to store multiple values in a single variable and access them efficiently using indices.

Array Declaration and Initialization

In C, arrays are declared by specifying the type of the elements and the number of elements the array will hold. The general syntax for declaring an array is:

        data_type array_name[size];

Here, data_type refers to the type of data (like int, float, char, etc.), array_name is the name of the array, and size indicates the number of elements the array will hold.

Example:

        int numbers[5];  // Declares an array of 5 integers

In the above example, the array numbers can hold 5 integer values.

Initializing Arrays

Arrays can also be initialized at the time of declaration. Here’s the syntax:

        data_type array_name[size] = {value1, value2, ..., valueN};

In this case, each element of the array is assigned a value at the time of initialization.

Example:

        int numbers[5] = {1, 2, 3, 4, 5};  // Initializes the array with 5 integer values

If the number of values specified is less than the size of the array, the remaining elements will be automatically initialized to 0.

Accessing Array Elements

Array elements can be accessed using their index. In C, array indices start from 0. To access an element, you use the array name followed by the index in square brackets.

Example:

        int first_value = numbers[0];  // Accesses the first element (1) of the array

Here, numbers[0] refers to the first element in the numbers array, which is 1.

Example C Program Using Arrays

Program: Find the Sum of Elements in an Array

        #include 
        
        int main() {
        int numbers[5] = {1, 2, 3, 4, 5};  // Declare and initialize the array
        int sum = 0;
        
        // Loop to calculate the sum of array elements
        for(int i = 0; i < 5; i++) {
            sum += numbers[i];  // Add each element to the sum
        }
        
        // Output the result
        printf("The sum of the array elements is: %d\n", sum);
        
        return 0;
        }

Output:
