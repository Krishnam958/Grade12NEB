Chapter 1: Data Base Management System

Data and Information

Data

Raw, unorganized facts that need to be processed. Example: Numbers, names, measurements.

Information

Processed data that is meaningful and useful. Example: "John scored 85% in Math."

Features of Information

  • Accuracy
  • Timeliness
  • Completeness
  • Relevance
  • Consistency

Database and Its Purpose

A database is an organized collection of structured data, typically stored electronically. The purpose is to store, manage, and retrieve data efficiently.

Database Terminologies

  • Table: A structure made of rows and columns to hold data.
  • Field: A column in a table that represents a data attribute.
  • Record: A row in a table; a set of related data.
  • Tuple: Another term for a record (row).
  • Object: An instance of a class in object-oriented databases.
  • Keys: Unique identifiers for records. Includes Primary, Foreign, Candidate keys, etc.

Data Dictionary

A centralized repository that stores metadata about data such as types, relationships, and constraints.

Introduction

A Database Management System (DBMS) is a software solution designed to efficiently manage, organize, and retrieve data in a structured manner. It serves as a critical component in modern computing, enabling organizations to store, manipulate, and secure their data effectively. From small applications to enterprise systems, DBMS plays a vital role in supporting data-driven decision-making and operational efficiency.

Characteristics/Features of DBMS

Computers have several features that make them versatile and efficient:

  • Data Modeling: Tools to create and modify data models, defining the structure and relationships within the database.
  • Data Storage and Retrieval: Efficient mechanisms for storing data and executing queries to retrieve it quickly.
  • Concurrency Control: Ensures multiple users can access the database simultaneously without conflicts.
  • Data Integrity and Security: Enforces rules to maintain accurate and secure data, including access controls and encryption.
  • Backup and Recovery: Protects data with regular backups and enables recovery in case of system failures.

Applications/Uses of DBMS

DBMS are widely used across various fields:

  • E-commerce and Business: DBMS handles vast amounts of data related to sales, inventory, customer information, and financial transactions, enabling efficient order management, product catalogs, and payment processing.
  • Banking and Finance:Banks rely on DBMS to manage customer accounts, transactions, loans, and credit card details.
  • Education: Educational institutions use DBMS to manage student records, course enrollments, grades, and library databases.
  • Healthcare: DBMS is crucial for storing and managing patient records, medical history, and treatment plans, facilitating efficient data access for healthcare professionals.
  • Manufacturing: DBMS helps manage inventory, track production processes, and manage supply chains.

Capabilities and Limitations of DBMS

Capabilities/Advantages/Merits/Pros of DBMS:

  • Improved data sharing and security.
  • Better data integration.
  • Uniform data administration.
  • Reduced application development time

Limitations/Disadvantages/De-merits/Drawback/Cons of DBMS:

  • Complex and costly
  • Requires skilled professionals
  • Hardware and software overhead

Types of Database Models

  • Hierarchical Model: Data is organized in a tree-like structure.
  • Network Model: Allows many-to-many relationships using graphs.
  • Relational Model: Organizes data into tables (relations) and uses SQL.
  • Entity-Relationship Model: Represents data using entities and their relationships.

Integrity Constraints and Types

  • Domain Constraint: Limits values in a column to a specific type/range.
  • Entity Constraint: Ensures each table has a unique primary key.
  • Referential Integrity: Maintains consistent relationships between tables using foreign keys.
  • Keys: Ensure uniqueness and relationships (Primary, Foreign, Candidate keys).

Normalization

Introduction

A process to organize data to reduce redundancy and improve data integrity.

Normal Forms

  • 1NF: Atomic values, no repeating groups.
  • 2NF: No partial dependency; in 1NF.
  • 3NF: No transitive dependency; in 2NF.

Advantages

  • Reduces data duplication
  • Improves consistency
  • Increases flexibility

Disadvantages

  • More complex queries
  • Increased number of tables

Centralized and Distributed Database

Introduction

Centralized databases store all data in a single location. Distributed databases store data across multiple locations but appear unified.

Advantages

  • Centralized: Easier maintenance, consistency.
  • Distributed: Improved reliability and speed for local users.

Disadvantages

  • Centralized: Single point of failure.
  • Distributed: Complex setup and synchronization issues.

Comparison

Centralized databases are simpler but vulnerable to downtime. Distributed systems are more complex but scalable and fault-tolerant.

Database Security

Introduction

Database security involves protecting the database from unauthorized access, corruption, or theft.

Challenges

  • Unauthorized access
  • SQL injection
  • Internal threats
  • Data leaks

Security Measures

  • Authentication and authorization
  • Encryption
  • Access controls
  • Regular backups

Roles of DBA

  • Manage users and permissions
  • Backup and recovery
  • Monitor performance
  • Ensure data security

Practical Topics

DDL and DML Languages

  • DDL (Data Definition Language): CREATE, ALTER, DROP
  • DML (Data Manipulation Language): INSERT, UPDATE, DELETE, SELECT

SQL Data Types

  • CHAR: Fixed-length character string
  • VARCHAR: Variable-length character string
  • BINARY / VARBINARY: Binary data storage
  • TINYBLOB / TINYTEXT: Small blob or text field
  • TEXT / LONGTEXT: Large text storage
  • ENUM: String object with a predefined value list
  • BIT: Stores bitfield values
  • TINYINT: Very small integer
  • BOOLEAN: TRUE or FALSE (alias of TINYINT(1))
  • INTEGER / INT: Standard integer
  • FLOAT / DOUBLE: Floating point numbers
  • DECIMAL: Fixed point number
  • DATE: Date (YYYY-MM-DD)
  • DATETIME: Date and time combination
